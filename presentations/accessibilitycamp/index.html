The problem

...or why Quail is needed for the modern web.

Temporal shift

The time between people publishing content that has an accessibility problem and when a report is received is a period when some users are sad.

Reporting != Action

Institutions like higher ed, or even corporations with different departments, find it difficult to conjole content authors.

Lack of context

Rich editing environments deserve immediate feedback .

Overview

  • A jQuery plugin - JavaScript
  • Comes with pre-defined guidelines WCAG 2.0 and Section 508.
  • Tests are easily defined, and Quail comes with over 200 tests.

Example code


$('#content').quail({ jsonPath : 'dist/resources',
                  guideline : 'wcag',
                  testFailed : function(event) { ... },
                  complete : function(event) { ... },
                });

Defining tests

From src/resources/tests.yml


imgHasAlt:
  type: "selector"
  testability: 1
  title:
    en: "Image elements must have an \"alt\" attribute"
  description:
    en: "All img elements must have an alt attribute"
  guidelines:
    508:
      - "a"
    wcag:
      1.1.1:
        techniques:
          - "F65"
          - "H37"
  tags:
    - "image"
    - "content"
  options:
    selector: "img"
    filter: ":not(img[alt])"
